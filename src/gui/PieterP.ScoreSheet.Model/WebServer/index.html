The web service API and documentation are currently a work in progress. At this moment, the API supports the following functions:

  • /api/matches.json (GET): Returns extensive information about the matches that are currently open in the application.
  • /api/points.json (GET): Returns the points of a certain match. This method expects two URL parameters:
    • MatchNr: The match ID (eg 'PVLBH06/003'). The score sheet with this match ID must be opened in the application.
    • MatchIndex: The match number (eg. 1 for the first match, 2 for the second match, etc.)
  • /api/points.json (PUT): Adjusts the points of a certain match. This method expects two URL parameters:
    • MatchNr: The match ID (eg 'PVLBH06/003'). The score sheet with this match ID must be opened in the application.
    • MatchIndex: The match number (eg. 1 for the first match, 2 for the second match, etc.)
    The request data must be a Json string that corresponds to the output from /api/points.json (GET).

ScoreSheet also comes with one simple visualization (/points.html). You can adjust this visualization or upload your own files (HTML, JavaScript, CSS, ...) in the directory $WWWDIR$
