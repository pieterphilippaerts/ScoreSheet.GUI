API documentation

ScoreSheet Web Service

The ScoreSheet web service exposes the score sheets that are currently open in the desktop application. It is intended for local integrations, live visualizations, and tools that need to read match metadata or update the point values of a specific individual match.

URL(s) $APIURLS$
Protocol HTTP only. HTTPS is not supported.
Authentication No authentication is required.

Configuration

The host and port are configured in the ScoreSheet GUI. By default, ScoreSheet listens on 127.0.0.1 using port 6221. The address 127.0.0.1 refers to the local computer, so the API is only reachable from that same computer.

To make the API reachable from outside devices, such as tablets or smartphones used to update scores, configure ScoreSheet to listen on all IP addresses assigned to the computer (including public addresses).

Windows permissions

ScoreSheet uses the HTTP server built into Windows to handle JSON requests. Programs without administrator privileges cannot listen on arbitrary HTTP URLs. To let ScoreSheet start the web service, either run ScoreSheet with administrator privileges, or have an administrator explicitly grant the user account the right to listen on the configured URL.

For the currently configured URL, an administrator can grant that right once by running this command in a console window: 

netsh http add urlacl url=$APIURL$ user=DOMAIN\user

Replace DOMAIN\user with the Windows account used to run ScoreSheet. Make sure to restart ScoreSheet after running the above command

The JSON service status is shown on the Web Service settings page. The same page also shows startup errors when the service cannot be started.

Response headers

Header Value Description
X-Powered-By ScoreSheet-<version> Returned by the built-in web server.
Access-Control-Allow-Origin * Allows browser-based clients to call the API from another origin.
GET /api/matches.json

Returns one object for each score sheet that is currently open in ScoreSheet. Each object includes team information, players, global match totals, and all individual matches with their point arrays.

Parameters

No parameters

Responses

Status Content type Body
200 application/json Array of JsonMatchResult objects.

Example request

GET /api/matches.json HTTP/1.1
Host: localhost:8080

Example response

[

GET /api/points.json

Returns only the points for one individual match on an open score sheet. The response is a flat JSON array containing two strings per set: first the home-side score, then the away-side score.

Query parameters

Name Required Type Description
MatchNr Yes String Match identifier of an open score sheet, for example PVLBH06/003. URL-encode special characters such as / when calling the endpoint from a client.
MatchIndex Yes Integer One-based individual match number on the score sheet. Valid values depend on the match system, commonly 1 through 5, 6, 10, 12, or 16.

Responses

Status Content type Body
200 application/json Array of strings with two entries per set.
400 text/html The client sent an invalid request when MatchNr is missing or empty.
404 text/html The requested item could not be found when MatchIndex is missing or invalid, the match number is not open, or the individual match index does not exist.

Example request

GET /api/points.json?MatchNr=PVLBH06%2F003&MatchIndex=1 HTTP/1.1
Host: localhost:8080

Example response

["11","7","9","11","11","5","11","8","",""]
Empty strings represent sets that have not been played or filled in yet. Score values can also contain special markers such as WO for walkover and FF for forfeit.
PUT /api/points.json

Replaces the point values for one individual match on an open score sheet. The request body must contain the same flat string-array shape returned by GET /api/points.json.

Query parameters

Name Required Type Description
MatchNr Yes String Match identifier of an open score sheet, for example PVLBH06/003.
MatchIndex Yes Integer One-based individual match number on the score sheet.

Request body

Type Required shape Description
string[] Exactly two entries for each set configured for the individual match. Entries are applied in pairs: left score, right score. For a best-of-five match, send ten values. Use empty strings for sets that should remain blank.

Responses

Status Content type Body
204 text/html Empty body. Returned after the score values have been accepted and applied.
400 text/html The client sent an invalid request when required parameters are missing or invalid, the body is empty, the JSON cannot be parsed as a string array, or the number of entries does not equal set count * 2.
404 text/html The requested item could not be found when the score sheet is not open or the individual match index does not exist.

Example request

PUT /api/points.json?MatchNr=PVLBH06%2F003&MatchIndex=1 HTTP/1.1
Host: localhost:8080
Content-Type: application/json

["11","7","9","11","11","5","11","8","",""]

Example response

HTTP/1.1 204 No Content

Schemas

Property names are emitted as shown below. In debug builds the JSON may be indented; in release builds it is compact.

JsonMatchResult
Property Type Description
MatchCountIntegerNumber of individual matches on the score sheet.
HomeClubId, AwayClubIdStringClub identifiers.
HomeTeam, AwayTeamStringTeam names.
StartHourStringScheduled start hour, typically formatted as HH:mm.
MatchNrStringMatch identifier used as MatchNr in points requests.
LevelIntegerCompetition level. Known values include 1 Super, 2 National, 3 Regional, and 4 Provincial.
SeriesStringSeries or division label.
Veterans, Men, Women, Youth, InterclubBooleanMatch category flags.
HomeDouble, AwayDoubleString or nullSelected double-player combination, for example 1/2, 1/3, or 2/3.
HomePlayers, AwayPlayersArray of JsonPlayerPlayers listed for each team.
MatchesArray of JsonMatchIndividual match details and points.
WonCount, LostCountIntegerCurrent score sheet match totals from the home-team perspective.
IsHomeFF, IsAwayFFBooleanWhether the home or away team is marked as forfeit.
JsonPlayer
Property Type Description
NameStringPlayer name.
ComputerNumberStringPlayer computer number.
PositionStringPosition on the strength list.
IndexStringPlayer index used by the score sheet.
RankingStringPlayer ranking.
CaptainBooleanWhether the player is marked as team captain.
IsWOBooleanWhether the player is marked as walkover.
JsonMatch
Property Type Description
MatchIndexIntegerOne-based individual match number. Use this value as MatchIndex for point requests.
HomePlayers, AwayPlayersArray of integersPlayer numbers used in the individual match. Doubles are represented as 0.
PointsArray of stringsFlat score array with two strings per set: home-side score, away-side score.
WonSets, LostSetsStringSet totals for the individual match from the home-side perspective.
WonMatches, LostMatchesStringRunning match totals at this point in the score sheet.
Point array

A point array is a JSON string array. For each set, the first value is the left or home-side score and the second value is the right or away-side score. The array length must match the number of sets configured for that individual match multiplied by two. 

[
  "11", "7",
  "9",  "11",
  "11", "5",
  "11", "8",
  "",   ""
]

Custom files

You can also host your own files through the ScoreSheet web server, including HTML, JavaScript, and CSS files. Put those files in the www directory in the ScoreSheet program folder. You can find the exact program folder location by starting ScoreSheet and opening the About ScoreSheet screen from the menu. 

$WWWDIR$

By default, this directory contains results.html, a visualization of the current status of the active matches. You can use it as an example for your own visualizations. It also contains edit_results.html which allows you to update scores from a browser interface.
